Монитор SmartComm — что есть

Локальный диагностический портал контроллера умного дома. Все данные собираются на самом контроллере и хранятся локально (без облака). Кликни на раздел чтобы раскрыть.

📊 Метрики контроллера (1-й ряд)
  • Температура CPU — текущая + спарклайн за час. Цвет: <60°C, 60-75, >75 критично. При перегреве плитка получает цветную рамку.
  • CPU — общая нагрузка %, частота, LoadAvg, мини-полоски по ядрам.
  • RAM / Диск / — % использования с полосой и спарклайном.
📡 Метрики MikroTik (2-й ряд, если настроен)
  • CPU — load %, частота, ядра, аптайм MikroTik.
  • RAM / Диск — % + использовано/всего + версия RouterOS.
  • WAN — скорость загрузки/отдачи через интерфейс с default-route, общий трафик. Кнопка WebFig открывает веб-интерфейс MikroTik.
🎛 Управление (3-й ряд)
  • iRidium сервис — статус (активен/загрузка/мёртв), PID, время запуска. Кнопки Портал, Рестарт, Старт/Стоп.
  • Uptime контроллера — кнопки 🖥 Терминал (WebSSH на :8022), Reboot, Shutdown.
  • Сеть eth0 — скорости и спарклайн трафика контроллера.
  • iRidium · детали — версия билда, имя проекта из .irpz, RAM/потоки, размер БД, число подключённых клиентов. Ссылка ⚙ API — настройка пароля iRidium.
📡 Мониторинг устройств

Плитка появляется если в Карте сети какие-то устройства отмечены галочкой «Показывать на дашборде». У каждого — 24-часовая полоска доступности и подпись «онлайн с DD.MM HH:MM». Офлайн сверху для удобства.

📈 Большие графики
  • Температура CPU контроллера + Нагрузка CPU — переключатель 1 ч / 24 ч.
  • MikroTik · CPU + MikroTik · WAN трафик (↓ download / ↑ upload в одном графике).
  • История в SQLite /var/lib/smartcomm-dashboard/metrics.db, переживает рестарты. 30 дней retention.
⚙️ Состояние системы (health-checks)

NTP синхронизация · упавшие systemd-юниты · нужен ли ребут · доступные apt-обновления (раз в 6 ч) · последние ошибки dmesg · SD-карта · throttle история (был ли undervolt/перегрев).

🗺 Карта IP (в самом низу)

Дубль карты сети с /network: 16×16 ячеек = вся подсеть /24.

  • Фон ячейки = тип адреса: зелёный static, жёлтый dynamic, коричневый шлюз .1, голубой = этот контроллер.
  • Точка справа-снизу = статус: зелёная online, красная offline.
  • 4 чекбокса-фильтра: static·on/off, dynamic·on/off. По умолчанию выключен dynamic·offline (давно отключённые устройства).
  • Шлюз и этот контроллер видны всегда независимо от фильтров. Клик → /network.
🎛 Управление в шапке
  • Карта сети — открыть страницу инвентаря.
  • 👥 Пользователи (только админу) — управление и аудит-лог.
  • О дашборде · Версия N.N.N · Кнопка Argon (только на Raspberry Pi с Argon ONE V3 кейсом).
  • Селектор обновления 1 / 5 / 15 / 30 с / 1 мин / выкл.
  • ⟳ обновить сейчас · ◐ тема (auto/light/dark) · 🔔 notifications · ⏻ выход.
📱 PWA + 🔒 Режим клиента

iPad/iPhone Safari: «Поделиться → На экран Домой» — иконка «Монитор SC» во весь экран.

URL /client — тот же дашборд без кнопок управления (read-only). Безопасно раздать клиенту/гостю.

Кнопка Argon ONE V3 · справочник

·

Короткое одиночное нажатие

Ничего не происходит. MCU кейса аппаратно фильтрует случайные касания.

··

Двойной клик (< 1 сек)

Reboot контроллера. Systemd корректно гасит iRidium и остальные сервисы перед перезапуском (~30 сек простоя).

━

Удержание 2–3 секунды

Безопасный shutdown. ОС корректно гасит всё, затем EEPROM-настройка POWER_OFF_ON_HALT=1 физически снимает питание. Включить — короткое нажатие.

━━

Удержание 5+ секунд

⚠ Жёсткий обрыв питания MCU кейса в обход ОС. Риск повредить базу iRidium и ФС — только при полном зависании.

Любое нажатие на выключенном контроллере

Включение. MCU подаёт питание, контроллер загружается за ~25 секунд.
