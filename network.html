Карта сети — что умеет

Инвентарь всех устройств в локальной сети с автодискавери, маркировкой, доступами, мониторингом доступности.

Кликни на раздел чтобы раскрыть.

🔍 Дискавери и опрос
  • 🔍 Сканировать сеть — nmap -sn -PR по всей подсети /24. Находит IP + MAC + вендор. ~30 сек на 100 устройств.
  • Автоскан — фон, по умолчанию раз в 4 ч (настраивается). Новые устройства попадают в очередь FP-опроса.
  • Presence check — ARP-скан раз в 60 сек + ICMP fallback (гистерезис 3 пропуска).
  • 🔬 Опросить службы — nmap -sV по 13 портам (22/80/443/554/1883/8080...). Заполняет открытые порты + URL веб-интерфейса.
  • 🔗 Sync MikroTik — имена устройств из DHCP-comments. Match по MAC, override always. Флаг static/dynamic. CP1251 декодируется автоматом. Раз в час auto-sync в фоне.
  • mDNS-обогащение — avahi-browse подтягивает «человеческие» имена (Sonos / Apple / HomeKit / принтеры).
  • 🏷 Переклассифицировать — пройти по «Без типа» и назначить тип по vendor (Hikvision→camera, MikroTik→network, и т.п.).
📋 Таблица устройств
  • Сортировка — клик по заголовку. IP сортируется натурально (.5 до .50). Повторный клик — обратный.
  • Фильтры: чипы Все / Онлайн / Офлайн + динамические per-type.
  • Поиск — имя / hostname / IP / MAC / описание / комната / заметки.
  • Колонка DHCP — static = закреплён за MAC, dynamic = любой свободный, «—» = не было синка с MikroTik.
  • 24h полоска в колонке «Статус» — мини-visualization доступности за сутки.
📝 Карточка устройства (клик на строку)
  • Поля: имя, помещение, тип, URL, описание, заметки.
  • ☑ Показывать на дашборде — устройство появится плиткой на главной с 24h-полоской.
  • Теги — Enter/запятая добавляют. Клик на × = удалить.
  • Фото — несколько на устройство, по 2 МБ. Hover показывает крестик удаления.
  • Доступы — несколько пар login/password (admin + viewer + RTSP и т.п.). 👁 показать, 📋 копировать.
  • История присутствия — uptime % за 30 дней + лента событий.
  • История изменений — audit log, retention 90 дней. Sync MikroTik помечается как sync_mikrotik.
  • Метаданные: IP / MAC / vendor / first_seen / last_seen.
  • Открытые порты — результат nmap -sV.
🔘 Действия в карточке
  • 💾 Сохранить — правки полей (теги, фото, доступы сохраняются автоматом).
  • ↗ Открыть веб-UI — если URL задан или подобран опросом.
  • 🖥 Терминал — SSH-сессия в браузере (через WebSSH на :8022).
  • 📋 SSH cmd — копировать ssh pi@<ip> в буфер.
  • 📡 Ping — ICMP + RTT мгновенно.
  • 🔬 Опросить службы — nmap -sV для одного устройства.
  • 📊 SNMP опрос — для типа network (нужно community).
  • 🗑 Удалить — убрать из карты (устройство не трогается).
☑ Bulk операции

Чекбоксы слева. При выборе всплывает нижняя панель: массово назначить помещение / тип / показ на дашборде. Удобно для «выбрать все камеры → присвоить помещение Двор» одним кликом.

🗺 Карта IP (переключатель справа)

Визуальная сетка 16×16 = 256 IP подсети.

  • Фон = тип: зелёный static, жёлтый dynamic, коричневый шлюз .1, голубой = этот контроллер.
  • Точка справа-снизу = статус: зелёная online, красная offline.
  • 4 чекбокса над картой. По умолчанию выключен dynamic·offline.
  • Шлюз и этот контроллер видны всегда независимо от фильтров.
  • Клик по ячейке — раскрытие карточки. Та же карта продублирована внизу дашборда.
